CI360 — Teste Web (Click + Identity + Email)

Click target ID: btn-ci360-evento-basico
Custom Event API key (JS API):
Identity var (User Identification): window.ci360_identity (customer_id)
Para Journey + Triggered Email: use o botão 3) Disparar Custom Event (JS API) (ele envia email_address junto).

Fluxo recomendado (melhor prática para validar Journey + Email):
1) Preencha customer_id e email_address.
2) Clique em 1) Set Identity (salva e envia ci360('identify')).
3) (Opcional) Recarregue (F5) 1x se sua identificação depende do Page View.
4) Clique em 3) Disparar Custom Event (JS API) para disparar o evento com email_address (isso destrava o Triggered Email).

Validação no browser: DevTools → Network → filtre por /t/s/ → veja POST 200/204.
Validação no CI360: use Diagnostics (Identity Map / Customer State) e a aba de eventos no Snowy.
