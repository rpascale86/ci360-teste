CI360 — Teste Web (Identity primária + login_id secundária)

Objetivo: replicar o comportamento do MOBILE no WEB:
customer_id como identidade primária + login_id (email) como identidade secundária.
Também enviamos attributes.email_address e attributes.email_permission para Journey/Triggered Email.

Fluxo recomendado:
1) Preencha customer_id e email.
2) Clique Set Identity → envia identify com customer_id + login_id (igual ao mobile).
3) Clique Disparar Journey → cria variáveis e dispara o gatilho de variável.
4) Verifique no CI360 Diagnostics / Event Stream e na Journey.
